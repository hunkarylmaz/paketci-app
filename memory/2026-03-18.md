## 03:00 - PostgreSQL vs MongoDB Karşılaştırması

**Soru:** Hangisi daha iyi, MongoDB mi PostgreSQL mi?

**Karar:** PostgreSQL kesinlikle daha iyi

| Kriter | PostgreSQL | MongoDB |
|--------|-----------|---------|
| Veri İlişkileri | ✅ FOREIGN KEY güçlü | ❌ Manuel referans |
| Finansal İşlemler | ✅ ACID %100 | ⚠️ Riskli |
| Raporlama | ✅ SQL güçlü | ❌ Aggregation zor |
| Kurye Sistemi | ✅ JOIN'lerle kolay | ❌ Çok sorgu gerekir |

**Proje PostgreSQL kullanıyor:** TypeORM + PostgreSQL kombinasyonu tercih edildi.

## 03:55 - API Entegrasyon Altyapısı

**Backend Modülü:** `integrations/`

### Adapter Pattern ile Platform API'leri
| Platform | Adapter | Özellikler |
|----------|---------|------------|
| Yemeksepeti | `yemeksepeti.adapter.ts` | Token yenileme, sipariş çekme |
| Getir | `getir.adapter.ts` | Otomatik auth, sipariş yönetimi |
| Trendyol | `trendyol.adapter.ts` | ⏳ Şablon hazır |
| Migros | `migros.adapter.ts` | ⏳ Şablon hazır |

### API Endpoint'leri
```
GET    /api/integrations/platforms
POST   /api/integrations/restaurants/{id}/platforms
PUT    /api/integrations/restaurants/{id}/platforms/{pid}/status
POST   /api/integrations/restaurants/{id}/platforms/{pid}/sync
```

### Güvenlik
- API anahtarları AES-256 ile şifreli saklanıyor
- Access token otomatik yenileniyor
- Webhook imza doğrulaması

### Veritabanı Şeması
```sql
platform_integrations          # Sabit platform listesi
restaurant_platform_configs    # Restoran-platform bağlantısı
  - api_key (ENCRYPTED)
  - api_secret (ENCRYPTED)
  - access_token, refresh_token
  - status: pending_setup/connected/disconnected/error
  - is_open, auto_accept_orders
```
